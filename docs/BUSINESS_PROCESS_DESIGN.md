## 1. 문서 개요

### 1.1 목적 및 범위

본 문서는 튀니지 e-People II 시스템의 **목표(TO-BE) 업무 프로세스**를 정의한 BPR 컨설팅 산출물이다.

**목적**:
- 현행(AS-IS) 업무 방식의 비효율과 문제점을 구조화하고, 시스템 도입 후의 목표(TO-BE) 업무 프로세스를 설계
- 상세응용설계(AA)와 데이터베이스 설계(DA) 결과를 반영하여 업무-시스템 간 정합성 확보
- PC1(시스템 개발업체)에게 업무 흐름과 시스템 기능 간의 관계를 명확히 전달
- 수원기관(BCRC/DGGPC)의 업무 재설계에 대한 합의 기반 마련

**범위**:
- 민원 접수·처리·완료 전 주기 프로세스
- 부패 신고 접수·조사·완료 프로세스 (민원과 완전 분리)
- 국민제안(e-Participation) 프로세스
- 헬프데스크(1881) 상담 프로세스
- BCRC/BRC 관리·감독 프로세스
- 외부 시스템 연계 프로세스 (ELISE, Tuntrust, DigiGo)
- 문서 관리(EDMS/CDMS) 프로세스
- SLA 관리 및 에스컬레이션 프로세스

### 1.3 현행(AS-IS) vs 목표(TO-BE) 비교 요약

| 구분 | AS-IS (현행) | TO-BE (목표) |
|------|-------------|-------------|
| **민원 유형** | 단일 유형(شكوى) | 5유형 체계 (G-01) |
| **접수 채널** | 웹 포털, 방문, ELISE(수작업) | 웹, 모바일, 이메일 자동, SNS, ELISE 자동 연계 |
| **처리기한(SLA)** | 일률 적용, 관리 미흡 | 유형별 SLA 자동 적용, D-Day 자동 추적, 에스컬레이션 |
| **이관** | 무제한, 사유 미기재 | 3회 초과 시 BCRC 자동 보고, 사유 필수 (G-02) |
| **다기관 협업** | 불가 (이관만 가능) | 공동처리 워크플로우 (G-03) |
| **기한 연장** | 비공식, 임의 처리 | 연장 신청→관리자 승인→자동 갱신→시민 통보 (G-04) |
| **국민 참여** | 없음 | 국민제안 게시판, 공감 기능, 채택 이행 추적 (G-05) |
| **부패 신고** | 민원과 혼재 | 완전 분리, DGGPC 전용, 익명 토큰 (원칙 1) |
| **통계·분석** | 기본 현황만 | 반복 민원·장기 미처리 분석, 기관 성과 공개 (G-06, G-11) |
| **알림** | 수동 통보 | SMS/Email/In-app 자동 발송, 수신 확인 추적 (G-10) |
| **담당자 부재** | 업무 방치 | 대리 처리 자동 전환 (G-09) |
| **지식 관리** | 없음 | 유사 선례 참조, 표준 템플릿 (G-07, G-08) |
| **인증** | ID/PW만 | ID/PW + PKI(Tuntrust) + DigiGo SSO |
| **언어** | 아랍어 중심 | 아랍어(RTL) + 프랑스어(LTR) 완전 지원 |

## 9. 문서 관리(EDMS/CDMS) 업무 프로세스

> **목적**: 민원·부패신고 처리 과정에서 생산·수집·보관되는 모든 전자문서의 생애 주기(Life Cycle)를
> 체계적으로 관리하고, 특히 부패신고 증거 문서의 기밀성·무결성을 보장하는 프로세스를 정의한다.
>
> **관련 기능**: 7.1.1~7.1.4 (EDMS), 7.2.1~7.2.4 (CDMS), G-14 (운영 매뉴얼 관리)
>
> **설계 원칙**:
> - EDMS는 일반 민원 처리 문서(공개/내부), CDMS는 부패신고 증거 문서(기밀)를 관리
> - 민원 Ticket과 문서의 자동 연계(N:M 관계)
> - 문서 무결성 검증(SHA-256 Hash), 버전 관리, 감사 추적(Audit Trail)
> - 보존 기간 만료 시 자동 폐기 워크플로우

## 10. SLA 관리 프로세스 (전체 통합)

> **목적**: 민원 유형별 법정 처리기한(SLA)의 설정·적용·모니터링·에스컬레이션을 체계적으로 관리하여
> 처리기한 준수율을 극대화하고, 기한 초과를 사전에 예방하는 통합 프로세스를 정의한다.
>
> **관련 기능**: 1.4.4 SLA 기준 관리, 3.1.6 처리기한 관리, 3.1.7 처리기한 연장 신청, 9.4 에스컬레이션 관리
>
> **관련 갭 기능**: G-01 (유형별 SLA), G-04 (기한 연장 신청), G-06 (장기 미처리 분석)

### 10.1 SLA 기준표

| 민원 유형 | 한국어 | 아랍어 | 법정 처리기한 | 연장 가능 | 최대 연장 |
|-----------|--------|--------|--------------|-----------|-----------|
| grievance | 고충민원 | شكوى تظلم | 60일 | 1회 30일 | 90일 |
| proposal | 제안 | اقتراح | 30일 | 1회 15일 | 45일 |
| inquiry | 질의 | استفسار | 7일 | 1회 7일 | 14일 |
| suggestion | 건의 | مقترح | 30일 | 1회 15일 | 45일 |
| report | 신고 | بلاغ | 15일 | 불가 | 15일 |

> **부패신고(CorruptionReport)**: 별도 SLA 체계 적용 (예비검토 15일 + 정식조사 90일)
> **국민제안(CitizenProposal)**: 심사 기간 60일 (별도 관리)

### 10.5 SLA 관리 비즈니스 규칙

| 규칙 ID | 규칙 | 상세 |
|---------|------|------|
| SLA-R-01 | 마감일 계산 | 접수 확인 완료 시점(status='received' 전환 시) + SLA 일수. 공휴일·주말 제외 옵션은 BPR에서 확정 |
| SLA-R-02 | 연장 신청 기한 | 기한 만료 D-3 이전까지만 연장 신청 가능. D-3 이후 연장 신청 불가 |
| SLA-R-03 | 연장 횟수 제한 | 유형별 최대 1회. 신고(report)는 연장 불가 |
| SLA-R-04 | 이관 시 SLA | 이관 시 잔여 기한 이월 (초기화되지 않음). 즉, 이관으로 기한을 늘릴 수 없음 |
| SLA-R-05 | 공동처리 SLA | 주관기관 기준 SLA 적용. 협조기관 회신 기한은 주관기관이 설정 (최대 SLA의 50%) |
| SLA-R-06 | 에스컬레이션 중복 방지 | 동일 민원에 대해 같은 레벨의 알림은 1회만 발송 |
| SLA-R-07 | 기한 초과 성과 반영 | 기한 초과 민원은 기관·담당자 성과 지표에 자동 반영. 월간 성과 리포트에 포함 |
| SLA-R-08 | 질의(7일) 특수 규칙 | SLA 7일 이하 유형은 D-7 알림 없이 D-3부터 에스컬레이션 시작 |
| SLA-R-09 | 부패신고 SLA 분리 | 부패신고는 민원 SLA와 완전히 분리된 체계 적용 (예비검토 15일 + 정식조사 90일) |
| SLA-R-10 | 집단민원 SLA | 묶음 처리(G-12) 민원은 대표 민원의 SLA 기준 적용. 개별 민원 SLA는 대표 민원과 동기화 |

## 11. 프로세스 성과 지표 (KPI) 체계

> **목적**: 시스템 전체의 업무 프로세스 성과를 정량적으로 측정·모니터링하기 위한
> 핵심성과지표(KPI) 체계를 정의한다. 각 프로세스별 KPI와 전사 KPI를 계층적으로 구성하여

### 11.2 전사 KPI 상세 정의

| KPI ID | KPI명 | 산식 | 목표값 | 측정 주기 | 데이터 소스 |

## 12. AS-IS vs TO-BE 상세 비교표

> **목적**: 현행 e-People(AS-IS)과 목표 시스템 e-People II(TO-BE) 간의 업무 프로세스 차이를
> 영역별로 상세 비교하여, 시스템 전환의 범위와 개선 효과를 명확히 한다.
>
> **활용**: PC1 개발업체 RFP 요구사항의 근거 자료, BCRC 이해관계자 설득 자료

### 12.1 민원 접수 프로세스 비교

| 비교 항목 | AS-IS (현행 e-People) | TO-BE (e-People II) | 개선 효과 | 관련 갭 |
|-----------|----------------------|---------------------|-----------|---------|
| 민원 유형 | 단일 "민원(شكوى)" 유형 | 5유형 분류 (고충/제안/질의/건의/신고) | 유형별 맞춤 처리, SLA 차등 적용 | G-01 |
| 민원 분류 | 비표준 분류 체계 | 대→중→소 3단계 Cascading 분류 | 기관 간 통계 집계 가능, 자동 배분 | G-01 |
| 접수 채널 | 웹 포털만 가능 | 웹 + 모바일 앱 + 이메일 + 1881 전화 | 접근성 확대, 디지털 소외 해소 | — |
| 인증 방식 | 단순 ID/PW | ID/PW + PKI(Tuntrust) + SSO(DigiGo) | 보안 강화, 사용 편의성 | — |
| 익명 신고 | 불가능 | 익명 토큰 발급 + 상태 조회 가능 | 부패신고 활성화, 신고자 보호 | — |
| 파일 첨부 | 기본 업로드 (제한적) | Drag & Drop + 다중 업로드 + 바이러스 스캔 | UX 개선, 보안 강화 | — |
| SLA 적용 | 모든 민원 동일 기한 | 유형별 차등 기한 (7일~60일) 자동 적용 | 처리기한 관리 정밀화 | G-01 |
| 다국어 | 프랑스어 중심 | 아랍어(RTL) + 프랑스어 완전 지원 | 아랍어 사용자 접근성 확보 | — |
| 접수 확인 | 수동 통보 또는 없음 | 자동 SMS + Email 즉시 통보 | 시민 체감 서비스 향상 | G-10 |

### 12.2 민원 처리(Back Office) 프로세스 비교

| 비교 항목 | AS-IS | TO-BE | 개선 효과 | 관련 갭 |
|-----------|-------|-------|-----------|---------|
| 민원 배분 | 수동 배분 (관리자) | 분류 코드 기반 자동 배분 + 수동 보정 | 배분 시간 단축, 오배분 감소 | — |
| 처리 워크리스트 | 목록 나열 수준 | D-Day 색상 배지 + 우선순위 정렬 + 필터링 | 담당자 업무 효율화 | G-04 |
| 이관 관리 | 횟수 제한 없음, 사유 미기재 | 횟수 카운터 + 사유 필수 + 3회 이상 BCRC 자동 보고 | 이관 남용 통제, 책임 소재 명확 | G-02 |
| 공동 처리 | 불가능 (단일 기관 처리) | 주관기관 + 협조기관 공동 처리 워크플로우 | 복합 민원 해결력 향상 | G-03 |
| 집단 민원 | 개별 처리 (중복 작업) | 묶음 처리 + 결과 일괄 통보 | 행정 효율화 | G-12 |
| 기한 연장 | 구두 협의 또는 무단 초과 | 시스템 연장 신청 → 관리자 승인 → 자동 갱신 | 투명한 기한 관리 | G-04 |
| 답변 작성 | 자유 형식 | 표준 템플릿 + 유사 선례 참조 | 답변 품질 균일화 | G-07, G-08 |
| 담당자 부재 | 민원 방치 | 부재 등록 + 대리자 자동 지정 | 처리 공백 방지 | G-09 |
| 결과 통보 | 포털 게시만 | SMS + Email + In-app + 수신 확인 추적 | 도달률 보장 | G-10 |

### 12.3 부패신고 프로세스 비교

| 비교 항목 | AS-IS | TO-BE | 개선 효과 | 관련 갭 |
|-----------|-------|-------|-----------|---------|
| 데이터 분리 | 민원과 동일 DB | 완전 분리 (별도 MSA 서비스) | 기밀성 보장, DGGPC 독립 운영 | — |
| 익명 신고 | 없음 | 익명 토큰 + 신원 보호 아키텍처 | 신고 활성화 | — |
| 조사 워크플로우 | 수동·비표준 | 5단계 체계화 (접수→예비→정식→조사→결과) | 조사 품질 향상 | — |
| 증거 관리 | 파일 단순 저장 | CDMS 기밀 문서 관리 (암호화+워터마크+감사) | 증거 무결성 보장 | — |
| 담당 조직 | 불명확 | DGGPC 전용 채널 + 270개 거버넌스팀 | 조직 체계 명확화 | — |

### 12.4 통계·분석 프로세스 비교

| 비교 항목 | AS-IS | TO-BE | 개선 효과 | 관련 갭 |
|-----------|-------|-------|-----------|---------|
| 통계 기능 | 기본 집계 (건수 수준) | 실시간 대시보드 + 추이 분석 + 지역별 시각화 | 데이터 기반 의사결정 지원 | — |
| 반복 민원 | 식별 불가 | 동일 시민/동일 사안 3회+ 자동 식별 + TOP 10 | 제도 개선 근거 발굴 | G-06 |
| 기관 성과 | 내부 참고 수준 | 73개 BRC 성과 공개 평가 + 시민 포털 게시 | 기관 간 선의의 경쟁 유도 | G-11 |
| 보고서 | 고정 양식 수작업 | 사용자 정의 보고서 + Excel/PDF 자동 생성 | 보고 업무 효율화 | — |
| KPI 체계 | 없음 | 4계층 KPI (사업목표→전사→기관→담당자) | 성과 관리 체계화 | G-11 |
