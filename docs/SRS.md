# SRS (Software Requirements Specification)
# e-People II — 튀니지 전자참여·부패방지 통합 플랫폼

**문서 버전**: v1.0  
**작성일**: 2025  
**작성자**: PMC 응용시스템 설계 전문가 (AA)  
**대상 독자**: PC1 현지 개발업체, Claude Code 개발 에이전트  

---

## 1. 시스템 개요

### 1.1 배경 및 목적

e-People II는 튀니지 총리실 산하 BCRC(중앙민원실)가 운영하는 국민 민원·참여 플랫폼의 2세대 시스템이다. KOICA ODA 사업(2024–2028, 총 600만 달러)을 통해 기존 e-People 모놀리식 시스템을 MSA(Micro Service Architecture) 기반으로 전환하고, 시민 참여 기능을 대폭 확장한다.

### 1.2 시스템 범위

| 구분 | 내용 |
|------|------|
| 사용자 규모 | 시민 다수 + 공무원 약 1,200명(73개 BRC) + BCRC/DGGPC 관리자 |
| 언어 | 아랍어(AR, RTL), 프랑스어(FR, LTR) |
| 접근 채널 | 웹 포털, 모바일 앱(WebApp), 이메일, SNS |
| 운영 환경 | CNI(튀니지 국가정보센터) Private Cloud + Kubernetes |

### 1.3 사용자 역할(Role) 정의

| 역할 코드 | 역할명 | 설명 |
|-----------|--------|------|
| `CITIZEN` | 시민 | 민원 신청, 부패 신고, 국민제안 |
| `ANONYMOUS` | 익명 신고자 | 익명 부패 신고 전용 |
| `BRC_OFFICER` | BRC 담당자 | 민원 처리, 답변 작성 |
| `BRC_MANAGER` | BRC 관리자 | BRC 내 민원 감독, 이관 승인 |
| `BCRC_ADMIN` | BCRC 관리자 | 전체 BRC 감독, 배분, 통계 |
| `DGGPC_OFFICER` | DGGPC 담당자 | 부패 신고 처리 |
| `DGGPC_MANAGER` | DGGPC 관리자 | 부패 신고 감독, 거버넌스팀 관리 |
| `SYS_ADMIN` | 시스템 관리자 | 전체 시스템 설정, 사용자 관리 |

## 2. 기능 요구사항 (Functional Requirements)

### 2.1 인증 및 계정 관리

#### FR-AUTH-001: 표준 로그인
- **설명**: ID/PW 기반 로그인
- **입력**: 사용자 ID, 비밀번호
- **처리**: 인증 성공 시 JWT 토큰 발급, 역할별 화면으로 리다이렉트
- **예외**: 5회 실패 시 계정 잠금 (30분)
- **UI 요소**: 로그인 폼, 비밀번호 표시/숨김 토글, 자동 로그인 체크박스

#### FR-AUTH-002: PKI 전자인증 로그인
- **설명**: ANCE Tuntrust Mobile ID / DigiGo 연계 인증
- **처리**: 외부 인증 서비스로 리다이렉트 후 콜백 처리
- **UI 요소**: PKI 인증 버튼, 인증 진행 중 스피너, 성공/실패 메시지
- **비고**: 프로토타입에서는 "PKI 인증 시뮬레이션" 버튼으로 구현

#### FR-AUTH-003: 익명 신고 토큰 발급
- **설명**: 부패 신고 시 실명 인증 없이 익명 토큰 발급
- **처리**: UUID 기반 일회성 토큰 발급 → 신고 추적용 코드로 활용
- **UI 요소**: 토큰 표시 화면, 복사 버튼, 저장 안내 메시지

### 2.2 시민 포털 (Front Office)

#### FR-CITIZEN-001: 포털 메인 화면
- **설명**: 시민이 접근하는 메인 대시보드
- **표시 요소**:
  - 빠른 메뉴 (민원 신청 / 부패 신고 / 국민제안 / 내 민원 조회)
  - 최근 공지사항 (최대 5건)
  - 민원 처리 현황 요약 통계 (전체 접수/처리 완료 건수)
  - 민원 처리결과 공개 게시판 (최근 10건)
  - 언어 전환 버튼 (AR / FR)

#### FR-CITIZEN-002: 민원 신청 (5단계 Step Wizard)

**Step 1 — 민원 유형 선택**
- 표시: 5개 유형 카드 (아이콘 + 제목 + 설명)
  - 고충민원(Réclamation): 권리 침해 구제 요청
  - 제안(Suggestion): 정책 개선 아이디어
  - 질의(Demande de renseignement): 법령·제도 문의
  - 건의(Doléance): 행정 개선 요청
  - 신고(Signalement): 위법 행위 신고
- 유형 선택 시 해당 처리 기한(SLA) 안내 표시

**Step 2 — 기관 및 분류 선택**
- 표시: 대분류 → 중분류 → 소분류 계단식(Cascading) 드롭다운
- 기관 자동 추천: 분류 선택 시 담당 BRC 기관명 자동 표시
- 직접 기관 검색 옵션 제공

**Step 3 — 내용 작성**
- 입력 필드:
  - 제목 (최대 200자)
  - 내용 (최대 2,000자, 글자 수 카운터)
  - 발생 지역 (행정구역 선택기)
  - 발생 일시 (날짜 선택기, 선택 사항)
- 실시간 유효성 검사

**Step 4 — 첨부파일**
- Drag & Drop 파일 업로드 영역
- 파일 목록 (이름, 크기, 삭제 버튼)
- 제한: 최대 5파일, 각 10MB 이하, 허용 확장자 표시

**Step 5 — 확인 및 제출**
- 입력 내용 전체 요약 표시 (수정 링크 포함)
- 개인정보 처리 동의 체크박스
- 제출 버튼 → 로딩 → 접수 완료 화면
- 접수 완료 화면: 민원 번호, 처리 예정일, 조회 방법 안내

#### FR-CITIZEN-003: 민원 목록 조회
- **표시**: 본인이 접수한 민원 목록 (카드 또는 테이블)
- **정렬**: 접수일 최신 순 (기본)
- **필터**: 상태별, 유형별, 기간별
- **검색**: 제목·민원번호 키워드 검색
- **페이지네이션**: 10건/페이지

#### FR-CITIZEN-004: 민원 상세 조회
- **표시 요소**:
  - 민원 기본 정보 (번호, 유형, 분류, 접수일, 처리기한)
  - 처리 상태 타임라인 (수직 스텝퍼)
  - 담당 기관 및 담당자 (이름 마스킹)
  - 처리 결과 (완료 시 표시)
  - 첨부파일 목록
  - 만족도 평가 (완료 시 5점 척도 버튼)
  - 이의신청 버튼 (완료 후 7일 이내)

**처리 상태 타임라인 예시**:
```
● 접수 완료       2024-03-01 09:30
● 분류·배분       2024-03-01 14:00  →  환경부 민원실
● 처리 중         2024-03-02 10:00
  └ [이관] 지역청 환경과로 이관  2024-03-03
○ 처리 완료       (예정: 2024-03-08)
```

#### FR-CITIZEN-005: 국민제안 게시판
- **목록 화면**:
  - 제안 카드: 제목, 분류, 작성자, 공감 수, 상태 배지
  - 정렬: 최신순 / 공감순 / 처리중 우선
  - 카테고리 필터
- **상세 화면**:
  - 제안 전문, 공감하기 버튼 (로그인 필요)
  - 기관 검토 의견 (채택/미채택 + 사유)
  - 이행 현황 업데이트 목록

#### FR-CITIZEN-006: 부패 신고 접수
- **경로**: 실명 신고 / 익명 신고 분기
- **익명 선택 시**: 토큰 발급 안내 → 토큰 저장 권고 → 신고 폼
- **신고 폼 필드**:
  - 신고 유형 (뇌물 / 횡령 / 직권남용 / 기타)
  - 피신고 기관
  - 발생 시기·장소
  - 신고 내용 (상세)
  - 증거 자료 첨부
- **제출 후**: 신고 접수 번호 발급, 진행 조회 방법 안내

### 2.3 공무원 Back Office

#### FR-BO-001: 담당자 메인 대시보드
- **표시 요소**:
  - 처리 대기 민원 건수 (긴급/일반/기한 임박 구분)
  - 오늘 처리 완료 건수
  - 이번 주 SLA 준수율
  - 처리 기한 임박 알림 목록 (D-1 이하)
  - 최근 배분된 민원 5건

#### FR-BO-002: 민원 워크리스트
- **표시**: 본인에게 배분된 민원 목록
- **컬럼**: 민원번호, 유형, 제목, 접수일, 처리기한, D-Day, 상태
- **D-Day 색상**: 
  - 7일 이상: 초록
  - 3~6일: 노랑
  - 1~2일: 주황
  - 초과: 빨강 + "기한 초과" 배지
- **필터**: 유형, 상태, 기한 임박 여부
- **일괄 처리**: 복수 선택 후 상태 일괄 변경

#### FR-BO-003: 민원 처리 화면
- **레이아웃**: 좌측 민원 내용, 우측 처리 패널 (2컬럼)

**좌측 — 민원 내용**
- 민원 기본 정보
- 시민 작성 내용 전문
- 첨부파일 미리보기
- 처리 이력 타임라인

**우측 — 처리 패널**
- 상태 변경 드롭다운
- 답변 작성 에디터 (리치 텍스트)
- 유사 선례 검색 버튼 → 사이드 패널 열기
- 표준 템플릿 선택 드롭다운
- 내부 협의 요청 버튼
- 이관 요청 버튼
- 기한 연장 신청 버튼
- 처리 완료 버튼

#### FR-BO-004: 이관 요청 화면
- **입력**: 이관 대상 기관 선택, 이관 사유 (필수, 최소 50자)
- **경고**: 이관 횟수 표시 (2회 이상 시 경고 메시지)
- **확인**: "이관 후 해당 기관으로 책임이 이전됩니다" 확인 모달

#### FR-BO-005: 유사 민원 선례 검색 (사이드 패널)
- **검색**: 현재 민원의 분류·키워드 자동 적용
- **결과**: 과거 유사 민원 카드 목록 (제목, 처리 결과 요약, 처리 기관)
- **적용**: 선례 선택 → 답변 에디터에 내용 복사 버튼

#### FR-BO-006: 기한 연장 신청
- **입력**: 연장 사유 (필수), 연장 요청 일수
- **처리**: 관리자 승인 대기 상태로 전환
- **승인 시**: 기한 자동 갱신, 시민에게 SMS/이메일 자동 발송

### 2.4 BCRC 관리자 기능

#### FR-ADMIN-001: 통합 현황 대시보드
- **표시 요소** (실시간 업데이트):
  - KPI 카드: 전체 접수 / 처리 완료 / 처리 중 / 기한 초과
  - 기관별 처리율 막대 차트 (Top 10 / Bottom 10 전환)
  - 일별 접수·처리 추이 꺾은선 그래프 (최근 30일)
  - 민원 유형별 파이 차트
  - 이관 다발 민원 경고 목록
  - 지역별 히트맵 (튀니지 지도 또는 표)

#### FR-ADMIN-002: 기관 성과 평가 화면
- **표시**: BRC 기관별 성과 테이블
- **컬럼**: 기관명, 접수, 처리완료, 처리율, 평균처리일, SLA준수율, 만족도, 이관 횟수
- **정렬**: 각 컬럼별 정렬 가능
- **하이라이트**: SLA 준수율 80% 미만 기관 빨간색 강조
- **공개 평가 버튼**: 평가 결과를 시민 포털에 공개

#### FR-ADMIN-003: 공통 코드 관리
- **기능**: 민원 분류 코드 대/중/소 CRUD
- **기관 관리**: BRC 기관 정보 등록·수정·비활성화
- **SLA 설정**: 민원 유형별 처리기한(시간) 설정

### 2.5 통계 및 분석

#### FR-STAT-001: 통계 메인 화면
- **기간 선택기**: 일/주/월/분기/연도, 사용자 정의 범위
- **차트 종류**:
  - 접수·처리 추이: 꺾은선 + 막대 혼합 차트
  - 유형별 분포: 도넛 차트
  - 기관별 성과: 수평 막대 차트
  - 지역별 분포: 테이블 또는 버블 차트
  - 만족도 추이: 꺾은선 차트
- **반복 민원 분석**: 동일 시민 반복 접수 패턴 테이블
- **장기 미처리**: 30일/60일 이상 미처리 민원 목록

#### FR-STAT-002: 보고서 내보내기
- **형식**: Excel (.xlsx), PDF
- **범위**: 현재 화면의 필터·기간 조건 그대로 적용
- **버튼**: "Excel 내보내기", "PDF 내보내기" (프로토타입은 버튼 표시만)

## 3. 비기능 요구사항 (Non-Functional Requirements)

### 3.1 성능

| 항목 | 목표 |
|------|------|
| 페이지 초기 로드 | 3초 이내 (3G 환경) |
| API 응답 시간 | 95% 요청이 2초 이내 |
| 동시 접속 사용자 | 500명 이상 지원 |
| 첨부파일 업로드 | 10MB 파일 30초 이내 |

### 3.2 보안

| 항목 | 요건 |
|------|------|
| 인증 | JWT + Refresh Token (Access: 1시간, Refresh: 7일) |
| 전송 암호화 | HTTPS (TLS 1.2 이상) 필수 |
| 개인정보 마스킹 | 화면 표시 시 주민번호·전화번호 마스킹 |
| 세션 타임아웃 | 30분 비활동 시 자동 로그아웃 |
| OWASP TOP 10 | 개발 전 과정에서 보안 코딩 가이드라인 준수 |
| 감사 로그 | 민원 열람·수정·삭제 모든 행위 로그 기록 |

### 3.3 접근성

| 항목 | 기준 |
|------|------|
| 웹 접근성 | W3C WCAG 2.1 Level AA 준수 |
| 색상 대비 | 4.5:1 이상 (텍스트) |
| 키보드 탐색 | 모든 인터랙티브 요소 Tab 접근 가능 |
| 스크린 리더 | ARIA 속성 적용 |

### 3.4 다국어

| 항목 | 요건 |
|------|------|
| 지원 언어 | 아랍어(AR), 프랑스어(FR) |
| RTL 지원 | 아랍어 선택 시 전체 레이아웃 RTL 전환 |
| 날짜 형식 | 언어별 지역화 (AR: هجري 또는 ميلادي, FR: DD/MM/YYYY) |
| 숫자 형식 | AR: 아라비아-인도 숫자 또는 서양식 선택 가능 |

## 8. 제약 사항

### 8.1 기술적 제약
- 오픈소스 소프트웨어 최대화 (COR-004 요건)
- 범용 개발 언어 사용 (Java, Python, PHP 등) — CNI 유지보수 가능성 고려
- CNI Private Cloud (K8s) 환경에서 컨테이너로 배포 가능해야 함
- 모든 소스코드는 Git 저장소로 관리

### 8.2 법적·제도적 제약
- 튀니지 개인정보보호법 준수 (INPDP 요건)
- ANSE 보안 승인(Homologation) 요건 충족
- ANCS 사이버보안 지침 준수

### 8.3 운영 환경 제약
- 튀니지 농촌 지역 3G 환경에서도 핵심 기능 동작
- 아랍어(RTL) 레이아웃 필수 지원
- 오프라인 상태에서 임시 저장 기능 필요 (선택)
